# /feature - Feature Pipeline

`/feature <request>` ÔÇö understand the client, research the repo through the deterministic `scan` tool (never reading source by hand), then plan + implement.
Heavy lifting delegates to `mustard-rt`; the orchestrator routes phases + emits events. You (the AI) do the reasoning that grain cannot: elicitation, decomposition, lapidation.

> **Progressive disclosure.** This file is the LIGHT path (most runs) + the shared ANALYZE. The Full-scope machinery ÔÇö DECOMPOSE, the PLAN materialisation, the wave wiring and the Full-only rules ÔÇö lives in **`../../../refs/feature/full-plan.md`**, read ONLY when scope detection returns `full`. Don't load it for a light task; that is the whole point of the split.

> **Artifact order.** The spec dir + `spec.md` + `meta.json` are MATERIALISED by `mustard-rt run spec-draft` (at scope detection below). They do **not** exist during research. The slug is born from `--intent` there. Never reference `.claude/spec/{slug}/spec.md` before it has been materialised.
