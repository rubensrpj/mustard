## Convention

- All cross-page visual components live in `src/components/page/{Name}/index.tsx` (folder-per-component, Wave 4 of spec `2026-05-23-dashboard-design-system`) and are barrel-exported from `src/components/page/index.ts`.
- Import path is always `@/components/page` — never import individual files directly. The barrel is auto-generated by `scripts/refactor-folder-per-component.mjs`.
- `KPICard` renders a stat card with a colored top-stripe accent; choose `KPIAccent` semantically: `emerald` good, `amber` caution, `rose` error, `indigo` primary, `violet` QA, `sky` info, `zinc` neutral.
- `PhaseChip` reads phase colors from `phaseTheme()` in `src/lib/phaseTheme.ts` — never hard-code Tailwind phase colors in components.
- `EventChip` reads event colors from `eventTheme()` — same file as `phaseTheme`.
- `EmptyState` has three variants: `info` (neutral), `warning` (amber), `error` (destructive). Accepts a `right` slot for action buttons.
- `DataCard` is a table/list wrapper — do NOT add `padded` unless the content is free-form (for tables, let table cells handle padding).
- Adding a new primitive: create folder `src/components/page/{Name}/index.tsx` → the barrel auto-picks it up on next codemod re-run. Component props: always export the `interface` alongside the component (same file).

## Domain features

- Domain components live under `src/features/{specs,workspace,economy,knowledge,prd,telemetry,amend,trace}/{Name}/index.tsx`.
- Pages import from `@/features/{name}` granular (`@/features/specs/SpecCard`) or aggregated (`@/features/specs`). Both work; both resolve via the per-feature `index.ts` barrel.
- Helper modules shared by 2+ components within a feature go to `features/{name}/_shared/{helper}.ts` (the underscore prefix marks "not a component" — the barrel skips it).
- `__tests__/` stays at the feature root (`features/amend/__tests__/`), not inside individual component folders.
- Never import a moved component from its old path. The codemod removed all `@/components/{domain}/{X}` paths; legacy references in pages must be rewritten before merge.

## Real examples in this codebase

- `src/components/page/KPICard/index.tsx` — accent stripe + value + label + hint.
- `src/components/page/PhaseChip/index.tsx` — uses `phaseTheme()` tokens as Tailwind classes.
- `src/components/page/EmptyState/index.tsx` — info/warning/error variants with right-action slot.
- `src/components/page/DataCard/index.tsx` — table/list wrapper with `padded` prop.
- `src/components/page/index.ts` — the barrel (auto-generated by the Wave-4 codemod).
- `src/features/specs/SpecCard/index.tsx` + `src/features/specs/_shared/spec-status.tsx` — domain component + colocated helper pattern.
